Step 01 of 05 · Foundations of input

Form Controls

The atoms an operator touches to steer the system — actions, filters, search, and the inputs behind every forecast and rate decision. One filled ember action per context; everything else stays neutral so the live number always wins the eye. Flip the theme top-right to verify each control in both modes.

Data Display

How the system shows what it knows. Every figure is mono and tabular so columns line up like a ledger; ember marks the one metric in focus while green and red stay reserved for directional data. 07 · Motion · how the system comes alive

Motion & light-up

Calm interface, deliberate moments. Controls settle quickly; charts earn one expressive beat — they ignite as they enter view: lines draw on, the confidence band fades up, and the live edge glows ember. Motion is reserved the same way color is — it points at the number that matters, then gets out of the way. Everything here honors prefers-reduced-motion.

Light-up & timing

The tokens and patterns behind every animated surface. Built on plain CSS + a tiny reveal observer — no animation library — so it ports cleanly and stays on-brand.

Principles
Reveal on view
Charts animate once when scrolled in — never on a loop that distracts from the data.
Ember moments only
